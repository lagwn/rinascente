# サイト制作構成案

## 1. 提案の基本方針
- 会社の信頼形成は Rinascentes コーポレートサイトで行い、製品理解と商談化は各製品サイトで行う分業構成とします。
- `会社` と `製品` の役割を分けることで、閲覧者の目的に合った情報導線を作ります。
- デザインルールとパーツを共通化し、制作効率と更新性を両立させます。
- 今回の本体提案には `CMS / 管理画面` と `YUMEHO / MICA30 共通の会員機能` を含めます。
- 会員認証エリアは Rinascentes 管理下に配置し、両製品で共通利用します。
- `オンライン決済` と `スレッド形式のコミュニティ機能` はオプションとし、本体は `60万円規模`、`全体3週間` の想定で整理します。

## 2. 全体構成の考え方
- Rinascentes コーポレートサイトは `信頼` と `ブランド発信` と `一覧性` を担う入口サイト
- YUMEHO は `比較検討`、`技術優位性の訴求`、`問い合わせ獲得` を担う主力サイト
- MICA30 は `製品説明`、`導入比較`、`問い合わせ獲得` を担う製品サイト
- 共通会員機能は、既存顧客向けのサポートと継続接点を担う
- CMS / 管理画面は、更新頻度の高い情報を内製で運用するための最小構成
- 全体を通して、最終的なアクションは `資料請求`、`お問い合わせ`、`見積相談`、`会員利用` に集約

## 3. サイトごとの役割

### 3-1. Rinascentes コーポレートサイト（11ページ）
- ブランドメッセージ、経営チーム紹介、沿革で企業の信頼を形成
- プレスリリース（4本）で事業の動きを発信
- 導入事例のクロスプロダクト紹介
- 製品サイトへ送客するハブとして機能
- 会員認証エリア（PHP 5ページ）を管理下に配置

### 3-2. YUMEHO 製品サイト（12ページ）
- トップページの構成:
  - ヒーロー（興味喚起 + 信頼バッジ）
  - 動画体験（世界観の訴求）
  - Why YUMEHO 3つの理由（安全性・運用性・継続性 + インパクト数字）
  - Technology（天井走行式デュアルレールの技術優位性 + 従来方式との比較ストリップ + 3パネル解説）
  - 想定導入施設
  - 全国導入マップ（12施設・6都府県）
  - ご利用施設の声
  - 中間CTA（デモ体験誘導）
  - ROI / 導入効果シミュレーション + コスト削減概算
  - 導入シミュレーションCTA（モックUI付き）
  - 開発者の声
  - FAQ（7問）
  - MICA30 クロスプロダクト紹介
  - 最終CTA
- 下層ページ: 製品紹介、導入シミュレーション（決済機能付き）、導入事例、導入フロー、価格・見積、補助金ガイド、FAQ、会社概要、お問い合わせ
- 導入シミュレーションでは施設タイプ・レール長・ハーネス追加数を選択し概算費用を算出

### 3-3. MICA30 製品サイト（9ページ）
- トップページの構成:
  - ヒーロー（信頼バッジ + 認証番号）
  - 課題提起（「その0.05mLが、患者の未来を変える。」）
  - カウンターストリップ（2024年認証 / 0.05mL / 8施設 / 8年）
  - MICA30について（3つのトピックカード）
  - 3つの理由（精密注入・安全設計・視認性）
  - 主要スペック
  - 想定使用施設
  - ご利用施設の声
  - デモ依頼CTA帯
  - FAQ（6問）
  - 全国導入マップ
  - 最終CTA
- 下層ページ: 製品詳細、ご利用施設の声、導入事例、FAQ、価格・導入、導入の流れ、会社概要、お問い合わせ

### 3-4. 共通会員機能（Rinascentes管理下）
- YUMEHO と MICA30 を共通アカウントで利用できる構成
- 設置方法、利用方法の動画を会員向けに提供
- 購入履歴を会員画面で確認できるようにする
- 製品導入後の接点を継続させる仕組み

### 3-5. CMS / 管理画面
- お知らせ、FAQ、一部固定情報を更新できる最低限の運用機能
- 会員向け動画の登録・更新
- 購入履歴の登録・更新
- 公開後の軽微な更新を制作会社依存にしすぎない構成

### 3-6. オプション機能
- スレッド形式のコミュニティ機能
- 両製品から利用できるオンライン決済機能

## 4. 情報設計のポイント
- コーポレートではブランドメッセージと経営陣で信頼を形成し、迷わせない
- YUMEHO では `興味喚起 -> 動画体験 -> 選ばれる理由 -> 技術的根拠 -> 実績 -> 声 -> ROI -> 不安解消 -> 行動` の順で構成
- MICA30 では `興味喚起 -> 課題提起 -> 実績数字 -> 製品概要 -> スペック -> 声 -> FAQ -> 行動` の順で構成
- 会員領域は `ログイン -> 動画 / 購入履歴` とシンプルに構成
- 主要CTAの文言はサイト全体で統一し、行動を選びやすくする
- 各サイト間はヘッダーのサイトスイッチャーで即座に遷移可能

## 5. 制作上の設計ルール
- 共通ヘッダー（サイトスイッチャー付き）、共通フッター、共通CTAパーツを設計
- 下層ページはテンプレート化し、更新しやすくする
- PCとスマートフォンの両方で読みやすい構成
- 製品ごとの差別化は色、写真、コピーで出しつつ、UIルールは統一
- スクロールアニメーション、マグネティックボタン、イメージリビールなど高品質なインタラクション
- フローティングCTA、スクロール追従ヘッダー（下スクロールで非表示）
- バックエンド機能は既存プロトタイプを活かし、全面再構築は避ける
- 管理画面は更新頻度の高い項目に限定し、短納期に対応
- 購入履歴は管理画面から登録・反映できる構成とし、決済連携は本体範囲に含めない

## 6. 本提案の前提条件
- 既存プロトタイプのデザインとHTML構成をベースにブラッシュアップ
- 原稿、製品情報、写真素材は支給または既存素材流用を前提
- CMS は `お知らせ / FAQ / 一部掲載情報 / 動画 / 購入履歴` の更新に絞る
- 会員機能は `共通ログイン`、`動画閲覧`、`購入履歴閲覧` に限定
- 購入履歴は管理画面からの手動登録を前提とし、自動連携は行わない
- スレッド機能は本体から切り分け、別見積り
- オンライン決済は本体から切り分け、別見積り

## 7. 3週間の制作スケジュール案

### 1週目
- 要件確定
- サイトマップ最終調整
- デザイン調整方針の確定
- 会員機能と管理機能の仕様整理

### 2週目
- 公開ページの実装調整
- CMS / 管理画面の導入
- 共通会員機能の実装
- 動画 / 購入履歴まわりの確認

### 3週目
- 全体結合テスト
- 表示確認
- フォーム / ログイン / 会員導線の動作確認
- 修正対応
- 公開準備

## 8. この構成にするメリット
- 会社情報と製品情報の役割が明確になる
- 製品ごとのターゲット（PT/OT、施設長、放射線技師など）に合わせた訴求ができる
- 共通パーツ化により、全体予算を抑えやすい
- 会員機能を共通化することで、両製品の運用負荷を下げやすい
- 既存プロトタイプを活かすことで、短納期に対応しやすい
- 高品質なインタラクションでブランド価値を訴求できる

## 9. 補足提案
- `オンライン決済` と `コミュニティ機能` は、公開後に利用状況を見ながら段階追加するのが安全です。
- YUMEHO の Technology セクションでは、競合（ゴムサスペンション方式）との差別化を6項目の比較ストリップと動画キャプチャ画像3枚で視覚的に訴求しています。
- 導入シミュレーションではハーネス追加数の選択に対応し、概算見積の精度を向上させています。
