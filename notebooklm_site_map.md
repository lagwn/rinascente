# 全体サイトマップ提案

## 1. この資料の前提
- 本提案は、プロトタイプの最終構成をもとに整理した制作提案です。
- 制作対象は `Rinascentes コーポレートサイト`、`YUMEHO 製品サイト`、`MICA30 製品サイト` の全3サイトです。
- 今回の本体提案には、公開サイトに加えて `CMS / 管理画面` と `YUMEHO / MICA30 共通の会員機能` を含めます。
- `決済機能` と `ファンミーティング的要素（スレッド形式）` はオプション扱いとします。
- 会員認証エリア（PHP）は Rinascentes 管理下に配置します。
- 本提案では、本体予算を `60万円規模`、スケジュールを `全体3週間` で整理します。

## 2. 全体像
- Rinascentes コーポレートサイトは、企業の信頼形成・ブランド発信・プレスリリースと製品サイトへの送客を担うハブサイトです。
- YUMEHO サイトは、情報提供、導入検討、問い合わせ獲得を担う主力サイトです。
- MICA30 サイトは、製品理解、導入比較、問い合わせ獲得を担う製品特化サイトです。
- 共通会員機能は、両製品の導入後フォローと継続接点づくりを担います。
- CMS / 管理画面は、公開後の軽微な更新を内製化するための運用基盤です。
- 全体の主要CVは `資料請求`、`お問い合わせ`、`見積相談`、`会員利用` です。

## 4. 各ページの役割

### Rinascentes コーポレートサイト
- Home: 企業の第一印象形成、ブランドメッセージ、経営チーム紹介、沿革、製品への送客
- Identity: 企業理念・ブランドストーリーの発信
- Press: プレスリリース一覧と個別記事
- Cases: 導入事例のクロスプロダクト紹介
- Contact: 総合窓口としての問い合わせ導線
- Login / Member: 会員認証と認証後ページ

### YUMEHO 製品サイト
- トップ: 製品価値の要約、動画体験、技術優位性（デュアルレール比較）、Why YUMEHO、導入実績マップ、利用者の声、ROI試算、開発者の声、FAQ、CTA
- 製品紹介: 安全性、構造、G-Suit、利用シーンの理解促進
- 導入シミュレーション: 条件に応じた構成提案と概算費用算出
- 導入事例: 施設タイプ別の説得材料
- 導入フロー: 導入までの流れを明確化
- 価格・見積: ROIシミュレーション、モデルケース、稟議支援
- 補助金ガイド: 利用可能な補助金制度と申請サポート
- FAQ: 比較検討時の障壁解消
- 会社概要: 企業信頼性の担保
- お問い合わせ: リード獲得の最終着地点

### MICA30 製品サイト
- トップ: 製品概要、課題提起（精密注入の重要性）、比較ストリップ、3つの理由、スペック、施設の声、FAQ、導入マップ、CTA
- 製品詳細: 仕様と機能の詳細説明
- ご利用施設の声: 3施設のテスティモニアル
- 導入事例: 導入後の活用イメージ形成
- FAQ: 導入前の疑問解消
- 価格・導入: TCO、導入形態
- 導入の流れ: 検討から導入までの可視化
- 会社概要: 企業信頼性の担保
- お問い合わせ: 資料請求と商談化

## 5. 主要導線
- Rinascentes コーポレートサイトから各製品サイトへ送客
- YUMEHO は `トップ -> 動画体験 -> Why YUMEHO -> 技術比較 -> 導入実績 -> 利用者の声 -> ROI -> お問い合わせ` を主導線に設定
- MICA30 は `トップ -> 課題提起 -> スペック -> 施設の声 -> FAQ -> お問い合わせ` の構成で比較検討を支援
- 両製品の既存顧客は `共通会員領域` から動画や購入履歴を確認できる
- 管理者側は `CMS / 管理画面` から必要情報を更新できる構成とする
- 各サイト間はヘッダーのサイトスイッチャーとフッターリンクで相互遷移可能

## 6. ページ数の整理
- Rinascentes コーポレートサイト: 6ページ + ニュース記事4本 + 認証後1ページ = 11ページ
- YUMEHO 公開ページ: 10ページ + 決済2ページ = 12ページ
- MICA30 公開ページ: 9ページ
- 共通会員認証: 5ページ（PHP・Rinascentes管理下）
- 公開ページ合計: 32ページ + 会員認証5ページ = 37ページ相当
- CMS / 管理領域: 1式
- 本体対象: `公開32ページ + 共通会員機能 + CMS / 管理画面`

## 7. 本提案の進め方
- 公開ページは既存プロトタイプの構成を活かして調整中心で進めます。
- CMS はフルスクラッチではなく、更新項目を限定した簡易管理方式で組みます。
- 会員機能は `共通ログイン / 動画閲覧 / 購入履歴確認` に範囲を絞ります。
- 購入履歴は管理画面側で手動登録・更新できる前提とします。
- 決済機能とスレッド機能はオプションとして切り分け、本体の短納期を優先します。
- デザインは全面刷新ではなく、既存トーンを活かしたブラッシュアップ前提とします。

## 8. まとめ
- 今回の本体提案は、`Rinascentes + YUMEHO + MICA30` の公開サイトに加え、`CMS / 管理画面`、`両製品共通の会員機能` まで含めた構成です。
- 本提案では、既存プロトタイプ流用と機能範囲の絞り込みを前提とします。
- `オンライン決済` と `スレッド形式のコミュニティ機能` は、公開後に追加しやすいオプションとして切り分けるのが適しています。
