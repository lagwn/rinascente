## プレゼン成功のための注意事項

### 資料の表示順序（必ずこの順番で）
1. 全体概要.pdf
2. ブラウザでデモ（Rinascentes → YUMEHO → MICA30 → 会員ページ）
3. sitemap 3枚（さっと見せる程度）
4. 概算お見積り.pdf → 別紙
5. サーバーセキュリティ等.pdf

### やるべきこと
- **デモは事前にブラウザで全ページ開いておく**（読み込み待ちは命取り）
- **シミュレーション機能は必ず実際に操作して見せる**（最大のインパクト）
- **「先ほどお見せした〜」と前パートを引用する**（つながりを感じさせる）
- **相手の名前で呼ぶ**（「真誠会様」ではなく担当者名）

### やってはいけないこと
- 全部の機能を説明しようとしない（要点だけ）
- 資料を読み上げない（資料は後で読めばいい、デモを見せる）
- 「個人事業ですが…」と自分から弱みを言わない（聞かれたら4層体制で答える）
- 技術用語を使いすぎない（CMS → 管理画面、WordPress → 更新システム）

### 想定Q&A と回答

| 質問 | 回答 |
|------|------|
| なぜ3サイトに分けるの？ | 閲覧者の目的が違うから。会社の信頼を知りたい人と、製品の詳細を知りたい人を同じページに送ると、どちらも中途半端になる。 |
| 他社と比較検討したい | もちろんです。ただ、**すでに動いているプロトタイプ**があるのは弊社だけです。他社は提案書からのスタートになります。 |
| 60万は高くないか？ | 3サイト32ページ + 会員機能 + 管理画面でこの金額は、相場の半額以下です。既存プロトタイプの活用で実現しています。 |
| 個人事業で大丈夫？ | 4層の安全対策（Xserverインフラ / CMS自己更新 / セキュリティ監視 / 外注パートナー）で対応が止まることはありません。そして私の本業はよろず支援拠点のコンサルです。消えることはありません。 |
| 医療認証サーバーは？ | 製品紹介サイトでは不要です。患者情報は一切扱いません。必要になった場合はAWSに移行可能です。 |
| 公開後の更新費用は？ | CMSがあるので基本的な更新は御社で可能。
| デザインの変更はできる？ | プロトタイプベースのブラッシュアップが前提です。大幅な変更は追加費用になりますが、現状のデザインは既にプロ品質で、スクロールアニメーション等も実装済みです。 |

## 最後に：このプレゼンで勝つための心構え

**「提案書を持ってくる制作会社」と「動くサイトを持ってくるコンサルタント」は、まったく別の存在です。**

今日のプレゼンの最大の武器は：
1. **すでに動いているプロトタイプ** — 言葉ではなく実物で語る
2. **導入シミュレーション** — 他社にはない具体的な機能
3. **公的機関のコンサルタントという信頼性** — 個人事業の不安を完全に上書きする
4. **公開後の運用まで見据えた提案** — 制作会社にはない視座

相手が30分後に感じるべきことは、ただ一つ。
**「この人に頼めば、間違いない。」**
