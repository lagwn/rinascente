# オプション概算見積り案

## 1. この資料の位置づけ
- 本資料は、本体見積りとは別で追加できるオプション機能の概算です。
- 対象オプションは `スレッド形式のコミュニティ機能` と `YUMEHO / MICA30 両製品からのオンライン決済機能` です。
- 金額は `税別` の概算です。

## 2. オプション1: ファンミーティング的要素

### 機能概要
- スレッド一覧
- スレッド詳細
- 投稿
- 返信
- 簡易的な管理機能

### 想定用途
- ユーザー同士の情報交換
- 活用事例や運用ノウハウの共有
- ファンミーティング的な継続接点の形成

### 概算金額
- `100,000円`

### 補足
- 画像投稿、通知、リアクション、通報機能などを追加する場合は別途上振れします。
- 本体と同時進行する場合も、確認工数が増えるためスケジュール延長の可能性があります。

## 3. オプション2: オンライン決済機能

### 機能概要
- YUMEHO サイトからの決済（導入シミュレーション結果からの遷移）
- MICA30 サイトからの決済
- 決済完了 / 決済キャンセルページ
- 申込情報と決済情報の紐付け
- 購入履歴への反映設計

### 想定用途
- 資料請求や問い合わせだけでなく、そのまま申込・決済まで完了できる導線づくり
- 製品ごとの販売導線の強化
- YUMEHO 導入シミュレーションからシームレスに決済へ進める体験

### 概算金額
- `100,000円`

### 補足
- 利用する決済サービス、決済フロー、通知要件により変動します。
- 購入履歴への自動反映や管理画面側の詳細管理を含める場合は、上限寄りになります。
- YUMEHO は既にプロトタイプに決済フロー（payment-success / payment-cancel）が実装済みのため、これを活用できます。

## 4. オプションを同時採用する場合の目安
- コミュニティ機能 + オンライン決済機能を同時に追加する場合
- 概算合計: `200,000円`
- 想定追加期間: `1週間程度`

## 5. 提案まとめ
- 本体は `公開サイト（3サイト32ページ）+ CMS / 管理画面 + 共通会員機能` を基本構成とし、オプションは段階追加の想定です。
- コミュニティ機能と決済機能は、利用目的が明確になったタイミングで追加するほうが、予算と開発リスクをコントロールしやすいです。
